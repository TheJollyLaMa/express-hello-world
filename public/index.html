I can authorize Cyclic access to a github repo and concurrently deploy my app with evry push to github. And here is the express-hello-world example from Jolly's Cyclic Fork deployed in the wild. (the link to the page you are currently on)

Jolly's Cyclic Deployment (https://zany-gold-shrug.cyclic.app/)

On the programmer's end, I navigate to cyclic.sh and it links with my github account through an OAuth gateway. Cyclic produces a Dashboard for me with the ability to deploy projects, a card library of all my deployed apps, and a list of any repositories it has access to that are not yet deployed. The user experience is pleasant and has a good aesthetic appeal. The Dashboard has a number of useful features, such as console output, transaction and error logging, environment management, and even Self-Hosting! When you deploy, your app is given a random cyclic.app url like the one above and now you can interact with your app like any other deployed app. But you also have that useful dashboard and a github repo, of course! With every git push, the server is updated.

By opening a local dev environment in my console, I get a rapid reload of my development progress through the use of Node Monitor (nodemon). Every save is displayed in my web browser at the express localhost port. When I get a fair amount of porgress made, or finally get something right after a long struggle, I push to my git repo and the update is almost immediately reflected on the globally deployed Cyclic url!

I see this as a wonderful opportunity to dovetail what I've been learning in node with my newly polished Python3 skills. Furthermore, I've identified github skills to be a necessity in this last year and this would really drive home some of those conepts, as well as allow me to learn more about the next level of collaborative software development by offering a well informed opinion to Kam and his Cyclic software. I would like to use Cyclic's serverless concurrent deployment architecture to attempt a deployment of Stealy Steve's app from a public git repo for all to see and learn about! The whole point of his software is to get junior devs to the showroom floor without all the many lines of infrastructure code that can be difficult and time consuming to understand and navigate!

With a little finess, luck, and skill, Stealy Steve may take his Corner Lot Stealer Global! By which I mean, the printout from the python program should be visible in the logs tab of my Cyclic Dashboard. And the project will have all the foundational infrastructure to develop browser based frontend as well as collaborate, edit, redeploy, and scale.
